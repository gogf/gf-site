## 发布说明规范

为保持各版本描述的一致性，建议新发布说明遵循以下规范和段落顺序：

1. **开头总结**：在 `## 主要变化` 之前的正文中，用一两句概括本版本的改进重点。
2. **`## 主要变化`**：包含统一的子标题，允许按顺序补充详细内容。
   - `### 语言`：语言层面的新特性、关键语法改变、类型系统更新等。
   - `### 工具链`：`go` 命令、构建/安装/模块工具或者诊断与质量工具的变化。
   - `### 运行时`：调度器、GC、编译器协作、调试能力及运行时性能改进。
   - `### 性能`：若存在独立的性能优化或编译器性能调整，可单列此节。
   - `### 标准库`：列举新增/改造的标准库包与 API。
   - `### 平台支持`：平台、架构、系统兼容性相关的新增说明。
3. **补充小节**：如需介绍测试、兼容性等内容，可在上述节之后以 `### 测试框架`、`### 兼容性承诺` 等形式补充。
4. **`## 参考资源`**：统一放在每篇文档末尾，用于链接官方发布说明或相关 Issue/Proposal。

以上结构已在现有历史文档中统一应用，后续手动编写或自动生成的版本说明请参照此模板，以保证阅读体验和对比分析的一致性。
